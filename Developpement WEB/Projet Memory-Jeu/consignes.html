PROJET : MEMORY

A RENDRE EN FIN DE TP :

1 fichier ZIP nommé MONNOM_monprenom.zip
Il doit contenir votre mémo de conception (au format .pdf) et l'ensemble des fichiers permettant de faire fonctionner votre projet (html, javascript, css, librairies).

Sujet :

Vous devez réaliser en JavaScript (et Jquery si vous le souhaitez) un jeu de MEMORY.

Contraintes du jeu

  • Le jeu est constitué plusieurs paires de cartes numérotées.
  • Les cartes sont présentées face cachée au départ du jeu.
  • En choisissant à chaque tour 2 cartes (qui sont alors retournées) le joueur dois essayer d'éliminer toutes les cartes. Si les 2 cartes sélectionnées sont identiques elles sont éliminées (trouvées) sinon elles restent en jeu (et sont à nouveau masquées).
  • Il est possible à tout moment de recommencer une nouvelle partie
  • Il existe un mode « normal » (où les cartes sont mélangées) et un mode « simplifié » (qui permet de terminer la partie très rapidement car les cartes ne sont pas mélangées)
  • Le score est calculé en continu tout au long de la partie. Il est constitué du nombre de tentatives (sélectionner 2 cartes est une tentative).
  • Votre programme devra indiquer clairement :
    ◦ Les cartes trouvées et les cartes non trouvées
    ◦ La fin de partie
  • Au minimum on devra pouvoir jouer une partie avec 10 paires de cartes mais vous gagnerez des points supplémentaires en proposant des options de jeu (et notamment le choix du nombre de carte à trouver).
  • Pour le développement de votre programme, vous devez impérativement utiliser les classes développées lors de la première séance (et disponible sur Moodle) : CodeMemory, ReponseMemory et Memory.

Critères d'évaluation :

  • Zip mal nommé => -10 points
  • Rendu en retard => -10 points/jour
  • Ne pas utiliser les classes CodeMemory, ReponseMemory et Memory => -10 points
  • Absence du mémo de conception => -1 points
  • Mémo de conception présent et sérieusement renseigné => 2 points
  • Application fonctionnelle (fonctionnalités minimales ok) => 8 points
  • Votre code (HTML,CSS et JavaScript) Clarté, simplicité et efficacité du code et des commentaires => 2 points
  • Votre ergonomie (compréhension immédiate du fonctionnement, informations claires sur l’état d’avancement de la partie) => 2 points
  • Design « soigné » (propre et efficace) => 2 points
  • Animations et/ou effets sonores à bon escient => 1 points
  • fonctionnalités complémentaires
    - pouvoir choisir le nombre de cartes est une option souhaitable
    - Vous pouvez imaginer d’autres options => 3 points
  • Point de notation complémentaire mérité (expliquer pourquoi en fin de dossier de conception) => X points

Note maximale : 20
